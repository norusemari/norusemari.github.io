깃 블로그 재설정

Posted Dec 20, 2020 2020-12-20T23:00:00+09:00 by norusemari

블로그 최초 설정 이후 아래 두 가지 문제를 해결하기 위해 블로그를 처음부터 재설정 하였고 관련 기록을 남김

문제 1. 좌측 CATEGORIES탭을 누르면 포스팅 폴더가 열리는데, 클릭시 404페이지가 열림.
문제 2. fork와 clone의 차이를 정확히 모르겠음.

1. Github jekyll-archives 플러그인 미지원 이슈

bundle exec jekyll s 명령어를 사용하면 jekyll 서버가 내 로컬 피씨에서 실행되고, 할당받은 웹주소를 통해 새로 수정된 코드가 블로그에 어떻게 반영되는지 볼 수 있다. 문제는 여기서 발생하는데 로컬에서 정상적으로 구동되기 때문에 문제가 없을 것을 기대하며 커밋/푸쉬를 하면 실제 블로그에서는 알 수 없는 에러가 발생한다.

내 경우에는 좌측에 있는 CATEGORIES탭을 누르면 포스팅의 카테고리가 폴더처럼 열리게 되어 있는데 해당 폴더를 클릭할 경우 404 에러 페이지가 열리는 문제가 있었다. 나름 구글링을 통해 원인을 찾아보았지만 결국에는 답을 찾지 못했고 친한 개발자 형에게에게 도움을 요청했다.

원인은 github에서 jekyll-archives 플러그인을 지원하지 않아서 발생하는 이슈였다. 로컬에서는 플러그인이 알아서 페이지를 생성해주지만 정작 github에서는 지원하지 않는 플러그인을 사용할 경우 예상치 못한 에러가 발생하게 되는것 이다.

1.1 해결방법

내가 개발에 대한 지식이 좀더 많았더라면 다양한 해결책을 찾았을 테지만 우선 가장 직관적이고 쉬운 방법을 택했다. 즉 레포지토리를 이원화 하여 한쪽에는 기존에 배운데로 커밋/푸쉬 하고, 여기서 나온 결과물만 덜어내서 다른 레포지토리에 커밋/푸쉬 하는 방법이다.

  • blog 레포지토리 <- 여기에 포스팅 글이나 변경된 내용을 반영하여 1차 커밋/푸쉬
  • norusemari.github.io 레포지토리 <- 1차 작업 결과물(로컬에 생긴 _site 폴더)을 2차로 커밋/푸쉬

2차까지 작업을 마무리 하면 내가 수정한 코드나 새로 추가한 포스팅이 로컬에서 본 결과와 동일하게 업데이트 되었음을 알 수 있다. 플러그인 미지원 이슈에 대한 개념을 알고난 뒤 관련 검색을 해보자 좀 더 많은 정보가 나왔는데 이는 다음에 적용해볼 예정이다.

2. fork와 clone의 차이

사실 이번에도 fork와 clone의 차이를 개념적으로 완벽히 공부한 것은 아니다. 하지만 잘 정리된 블로그를 통해 필요한 개념은 익혔고, 로컬에 있는 저장소와 git에 있는 레포지토리를 여러번 삭제하고 재생성 하는 과정을 통해 두 방법을 모두 사용해 보았다. 현재는 clone을 통해 블로그를 셋팅해 두었다.

2.1 fork 개념 및 사용 예시

  • fork 개념 : 다른 사람의 Github repository에서 내가 어떤 부분을 수정하거나 추가 기능을 넣고 싶을 때 해당 respository를 내 Github repository로 그대로 복제.[출처]
  • fork 사용 예시 : 원하는 테마를 내 깃허브안에 복사해서 사용한다.

내가 선택한 테마 레포지토리 우상단에 있는 fork를 클릭하면 내 깃허브 안에 포크된 레포지토리가 생성된다. [user id]/[repository name] 형태로 레포지토리가 생성되는데 셋팅에서 rename을 통해 레포지토리 이름을 변경해서 사용할 수 있다. 다만 포크한 테마 레포지토리를 이름만 바꿔서 사용 할 경우 잔디가 반영되지 않는다고 하니 fork를 할지, clone을 할지는 본인의 선택.

2.2 clone 개념 및 사용 예시

  • clone 개념 : 특정 repository를 내 local machine에 복사하여 새로운 저장소를 만든다.[출처]
  • clone 사용 예시 : 깃허브에 있는 레포지토리를 내 로컬에 새로운 저장소로 내려받는다.

내 경우에는 아래 순서에 따라 테마를 적용하였고 4번 단계에서 clone 사용

  1. 테마 원본 레포지토리에서 Code 버튼 > Download ZIP 버튼을 클릭하여 테마 소스파일 다운
  2. 내 깃허브에서 noru.blog 레포지토리 생성
  3. mac terminal 창에서 로컬 저장소를 만들 위치로 이동(cd 명령어 사용)
  4. 명령어 git clone https://github.com/norusemari/noru.blog.git 입력
    ㄴ 초기화/git init 까지 같이 진행되기 때문에 로컬 저장소에 .git 폴더도 만들어져 있음
  5. 만들어진 로컬 레포지토리 안에 다운 받은 테마소스를 붙여넣기.
    ㄴ 테마에 따라 가이드에 맞춰 일부 파일/폴더 삭제 후 붙여넣는다.
  6. 테마 가이드에 맞춰 초기 셋팅 후 커밋/푸쉬
