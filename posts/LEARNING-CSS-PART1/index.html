3.0 How to Add CSS to HTML

css를 html 페이지에 추가하는 방법은 두가지가 있다.

  1. 같은 html 파일에 html코드와 css코드를 함께 작성하는 방법 : head tag 안에 style tag를 작성하고, style tag안에 코드를 작성.
  2. html와 css파일을 분리해서 작성하는 방법. 여러곳에 css파일을 범용적으로 이식하여 사용할 수 있으므로 이방법을 추천.

    • href=”style.css” : href attribute를 사용해 어떤 css파일을 연결할지 지정.
    • rel=”stylesheet” : rel attribute를 사용해 css파일과 이 html문서간의 관계가 stylesheet임을 지정.

3.3 Blocks And Inlines

박스들의 경우 어떠한 박스를 만들더라도 옆에 오지 않고 아래쪽으로 종으로 이어서 만들어진다. 그게 div, header, main, section, footer, article, address 등등 어떤 박스던지 모두 동일하다. 반면에 span, a, link, img, code 와같은 tag 들은 박스가 아니기 때문에 여러개를 이어서 작성하면 횡으로 이어져서 출력된다.

  • 옆에 다른 요소가 올 수 없는 tag를 block이라 하고
  • 옆에 다른 요소가 올 수 있는 tag를 inline이라 한다

대부분의 box들은 block이고 옆에 아무것도 올 수 없다. 따라서 block이 아닌 tag들을 아는게 더 효율적이다.

흰색 박스(div tag)에만 margin 값을 적용하고 싶은데 위의 코드에서는 상단 margin을 바꿔도 검은색 body 와 흰색 div가 같이 움직인다. 이러한 현상을 collapsing margin 이라 한다. 크롬에서 F12 단출키를 통해 요소검사를 해보아도 margin은 코드가 명명한데로 잘 적용되어 있으나, 욉에서 보여지는 모습은 내 의도대로 움직이지 않고 있다.

collapsing margin에 대해 다시 설명해보자면 안에 있는 박스의 경계가, 밖에 있는 박스의 경계와 같다면 이 두 박스 하나로 취급되는 것을 뜻한다. 그리고 이 현상은 상/하 margin 조정시에 발생한다. 또한 이러한 경우를 해결하기 위해 padding tag가 존재한다.

  • 마진 상쇄(collapsing margin)에 대해 잘 정리한 게시물 링크
  • collapsing padding에 대해 이해하고 외우려 하지마라!! 박스간의 마진 조정시 이런 현상이 일어나면 아.. 이게 collapsing padding 이었지 라고 떠올리면된다. 깊게 공부하는건 시간낭비.

위에서 눈여겨볼 포인트는 두가지. 첫 번째, style 안에서 특정 element를 가리키는 방법으로 #id_name을 사용할 수 있다. 두 번째, 공통된 element에 대해 스타일을 지정할때는 특정 element를 지정할 필요 없이 공통되는 element를 통해 지정해주면 된다.

  • 첫 번째는 위의 코드에서 #first{}, #second{}, #third{}, #fourth{}라고 적힌 부분을 살펴보자. element 명인 div를 사용하지 않고 각 div의 id를 호출해서 직접 스타일을 지정했다.
  • 두 번째는 style 안에서 div{}를 살펴보자. body안의 모든 div에 대해 padding : 10px 값을 지정해 주었다.

#tomato : id = “tomato” .tomato : class = “tomato”

또한 tag하나당 하나의 값을 가져야 하는 id와는 달리 다음과 같이 하나의 tag에 여러개의 클래스 이름을 가질 수 있다.

  <span>hello</span>
  <span class="tomato hello honey potato">hello</span>
  <span>hello</span>
  <span class="tomato">hello</span>
  <span>hello</span>
</body>

3.9 Inline block

<!DOCTYPE html>
<html lang="kr">
  <head>
    <title>Home - my first website</title>
    <style>
      body {
        margin: 20px;
      }
      div {
        display: inline-block;
        width: 50px;
        height: 50px;
        background-color: teal;
      }
    </style>
  </head>
  <body>
    <div></div>
    <div></div>
    <div></div>
  </body>
</html>

box형식인 div를 스타일에서 display: inline-block; 으로 설정해주면 box 형식인 div가 inline 형식으로 변환되면서 횡으로 배치된다. 그러나 inline-block은 다음과 같은 단점들이 있다.

  • 단점 1: margin, padding, border 를 따로 지정하지 않았음에도 div사이에 공백이 생겨있음
  • 단점 2: 또한 inline-block은 Responsive Design(반응형 디자인)을 지원하지 않는다. 따라서 사용자에 따라 창 크기가 달라지면 영향을 받는다.

따라서 inline-block과 같은 스타일은 요즘에 잘 사용하지 않는다. 개념상 이런게 있다는 정도만 알아두자.
