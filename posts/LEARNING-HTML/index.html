Learning HTML

Posted Jan 3 2021-01-03T11:00:00+09:00 by norusemari
Updated Jan 3 2021-01-03T17:08:06+09:00

html은 content가 무엇인지를 정해주고
css는 content들이 어떻게 보일지를 정해준다.
javascript는 웹사이트를 똑똑하게 동적으로 보여준다.

  • html : markup language, markup은 content
  • css : 디자인과 스타일을 위한 언어
  • javascript는 프로그래밍 언어이지만 html, css는 프로그래밍 언어가 아니다!

      href="https://nomadcoders.co/m.png"

    <meta charset="utf-8" />

  </head>
  <body>
    <h1>Hello!</h1>
    <a href="http://google.com" traget="">Go to google!</a>
    <img src="noru.jpg" />
    <br>
    <p>My name is <abbr title="Noru semari">Noru</p>

  </body>
</html>
  • meta element: The meta element represents various kinds of metadata that cannot be expressed using the title, base, link, style, and script elements.
  • head tag 안에는 페이지에서 보이지 않는 여러가지 설정들을 해줄 수 있다. meta역시 그러한 설정을 위한 tag중 하나.
    • meta 태그들은 self-closing tag
  • meta charset=”utf-8” : 브라우저에게 text를 어떻게 작성할지 설정
    • 한글이나 특수문자가 있는 언어 사용시 브라우저가 이해를 못하는 경우가 있음
    • 따라서 이 charset 메타 태그가 없으면 글자가 깨질 수 있음. 따라서, 꼭 넣어주길!!
  • html 태그의 lang attribute은 해당 페이지의 주언어가 무엇인지 검색엔진에게 알려주는 역할을 함.
  • link rel=”shortcut icon” 코드를 통해 웹페이지 상단탭에 작은 이미지를 넣어줄 수 있다.


  • meta tag > property=”og:~”
    • 카톡과 같은 메신저에서 링크 공유시 나타나는 이미지, 제목, 설명 등에 대한 설정.

  • The form element represents a collection of form-associated elements, some of which can represent editable values that can be submitted to a server for processing.
  • form tag에 대한 내용이고 그중에서도 안에 작성되는 input tag와 attribute들에 대해 눈여겨 보자.
  • input tag에서 type attribute이 무엇이냐에 따라 단순 텍스트 입력창이 될수도 있고, 전송, 파일 제출 버튼이 될수도 있다.
  • 모든 tag와 attribute들을 외울 수 없다. 기본 문법 형태만 알면 나머지는 찾아서 사용하면 된다.

        <label for="Profile">Profile Photo</label>

      </div>
      <div>
        <label for="website">Web Site</label>
        <input id="website" required placeholder="url" type="url" />
      </div>
      <input type="submit" value="Create Account" />
    </form>
  </body>

전반적으로 html문서에서 가장 많이 보이는 태그는 div tag는 division(분할/구분/경계선)이라는 단어에서 기원되었으며 일종의 박스라고 생각하면 된다.

  • div tag로 감싼 content들은 양옆이 아니라 종으로 배치된다.
  • 의미론적으로 봤을때 div tag는 document 안에서 아무런 의미가 없는 tag.
    • head, body, main, footer 등은 기능적으로는 div와 동일하지만 코드 자체가 의미를 가지고 있으며 이런 코드를 semantic 코드라 함.
  • 문서를 보기만 해도 그 의미를 짐작할 수 있는 것을 semantic이라 한다.

  • head, main, footer tag 모두 div tag로 변경해도 웹페이지에 나오는 결과는 동일하다.
  • div tag를 사용하고 id attribute으로 구분해주는 것도 방법이겠지만 위와 같이 프로그래머가 코드를 읽었을때 어떤 영역인지를 직관적으로 이해할 수 있는 tag가 더 좋다. 이는 검색엔진이 정보를 읽어들일때도 마찬가지.
  • 따라서 코드를 semantic 하게 짜도록 노력하자. document와 content를 명확하게 묘사할 수 있도록!
